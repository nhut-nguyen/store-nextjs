GearHub Store

Hướng dẫn kỹ thuật build và chạy dự án trên máy PC Windows

Tài liệu này dùng cho dự án Next.js bán máy tính, laptop, điện thoại và phụ kiện. Nội dung bám đúng cấu trúc hiện tại của project, bao gồm cấu hình SQL Server, build production và các bước kiểm tra sau khi deploy.

1. Mục tiêu

Đưa project từ môi trường phát triển sang máy Windows để:

  • cài đủ môi trường Node.js
  • cấu hình kết nối SQL Server
  • khởi tạo database PRO_STORE_NEXTJS
  • build ứng dụng production
  • chạy ứng dụng ổn định trên Windows

2. Yêu cầu môi trường

Phần mềm bắt buộc

  • Windows 10 hoặc Windows 11
  • Node.js 20 LTS hoặc mới hơn
  • npm đi kèm Node.js
  • SQL Server đang chạy và cho phép truy cập qua TCP/IP

Khuyến nghị

  • Git for Windows
  • PowerShell hoặc Windows Terminal
  • Visual Studio Code
  • SQL Server Management Studio để kiểm tra DB
IP Database hiện tại: 192.168.1.116
Database mặc định: PRO_STORE_NEXTJS
User mặc định theo project: sa

3. Chuẩn bị source code trên Windows

  1. Copy toàn bộ source code dự án sang máy Windows hoặc clone từ repository.
  2. Mở thư mục dự án bằng PowerShell hoặc Windows Terminal.
  3. Kiểm tra Node.js và npm:
node -v
npm -v

Nếu chưa có Node.js, cài từ bản LTS chính thức rồi mở lại terminal.

4. Cài dependencies

npm install

Lệnh này sẽ cài các package cần thiết như next, react, mssql, zod, recharts.

5. Tạo file cấu hình môi trường

Tạo file .env.local ở thư mục gốc dự án với nội dung:

SQLSERVER_USER=sa
SQLSERVER_PASSWORD=sa@123
SQLSERVER_HOST=192.168.1.116
SQLSERVER_PORT=1433
SQLSERVER_DATABASE=PRO_STORE_NEXTJS
SQLSERVER_ENCRYPT=false
NEXT_PUBLIC_APP_NAME=GearHub
Lưu ý: nếu password của SQL Server trên máy Windows khác sa@123 thì phải thay lại đúng giá trị thực tế.

6. Chuẩn bị SQL Server

6.1. Điều kiện cần

  • SQL Server phải bật chế độ SQL Authentication.
  • Tài khoản sa phải đăng nhập được.
  • TCP/IP phải được bật trong SQL Server Configuration Manager.
  • Port 1433 phải mở hoặc đúng port đang dùng.

6.2. Kiểm tra nhanh kết nối

Có thể dùng SSMS đăng nhập với:

  • Server: 192.168.1.116
  • Authentication: SQL Server Authentication
  • Login: sa
  • Password: mật khẩu thực tế

7. Khởi tạo database cho project

Dự án có sẵn script tạo lại schema và seed dữ liệu:

npm run db:init

Lệnh này sẽ:

  • tạo database PRO_STORE_NEXTJS nếu chưa tồn tại
  • đọc [database/schema.sql] và [database/seed.sql]
  • xóa bảng cũ rồi dựng lại toàn bộ schema
  • nạp dữ liệu seed ban đầu
Cảnh báo: npm run db:init là lệnh reset dữ liệu theo seed. Không dùng trên môi trường đã có dữ liệu thật nếu bạn chưa backup.

8. Chạy ở chế độ development

npm run dev

Sau khi chạy thành công, truy cập:

  • http://localhost:3000

Nếu muốn kiểm tra nhanh:

  • Trang chủ storefront
  • Trang sản phẩm /products
  • Trang admin /admin
  • API danh mục /api/categories

9. Build production trên Windows

9.1. Build

npm run build

9.2. Chạy production

npm run start

Mặc định ứng dụng sẽ chạy trên port 3000.

9.3. Nếu cần đổi port

PowerShell:

$env:PORT=3001
npm run start

Command Prompt:

set PORT=3001
npm run start

11. Các lỗi thường gặp và cách xử lý

Lỗi không kết nối được SQL Server

  • Kiểm tra lại SQLSERVER_HOST, SQLSERVER_PORT, user và password.
  • Đảm bảo SQL Server đã bật TCP/IP.
  • Kiểm tra firewall Windows có chặn port 1433 không.

Lỗi khi chạy npm run db:init

  • Đảm bảo tài khoản có quyền tạo database.
  • Kiểm tra file .env.local đã đúng chưa.
  • Kiểm tra SQL Server có đang chạy không.

Lỗi khi chạy npm run build

  • Chạy lại npm install.
  • Xóa thư mục .next rồi build lại.
  • Kiểm tra version Node.js, nên dùng Node.js 20 LTS trở lên.

App lên nhưng không có dữ liệu

  • Chạy lại npm run db:init.
  • Kiểm tra kết nối DB có trỏ đúng PRO_STORE_NEXTJS không.
  • Nếu SQL Server chưa sẵn sàng, app có thể fallback sang dữ liệu mẫu trong lib/data.ts.

12. Khuyến nghị khi bàn giao cho máy Windows

  • Khóa version Node.js dùng chung giữa các máy triển khai.
  • Không commit file .env.local chứa thông tin thật lên repository công khai.
  • Nếu chạy production lâu dài, nên dùng reverse proxy hoặc service manager.
  • Nếu database là máy riêng trong LAN, nên kiểm tra ping và quyền truy cập trước khi build.
Quy trình khuyến nghị ngắn gọn:
npm install → tạo .env.local → npm run db:init → npm run build → npm run start

File này được tạo riêng cho project GearHub Store để hỗ trợ build/chạy trên máy Windows.
